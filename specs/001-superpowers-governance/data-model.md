# Governance Surface Model

## Governance Contract

| Field | Meaning |
|---|---|
| authority | `github/spec-kit` for scope, approval and lifecycle |
| execution_discipline | `obra/superpowers` for the workflow inside an authorized step |
| conflict_rule | Spec Kit wins on scope and gates; pause and record a real conflict |
| amendment | Explicit owner request plus constitutional update |

## Execution Skill Policy

| Field | Meaning |
|---|---|
| automatic | A relevant skill is selected without manual activation |
| task_matched | The skill runs only when its declared area applies |
| capability_aware | Missing platform support is recorded and handled safely |
| lifecycle_gated | Implementation skills wait for the corresponding Spec Kit gate |

## Reference Surface

| Field | Meaning |
|---|---|
| local | `AGENTS.md`, constitution and policy are normative local records |
| reference | Root and project documents link back to the canonical contract |
| remote | Mem0 preserves a tagged governance memory |
